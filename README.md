# Flash9 ⚡

极简 · 临时 · 即焚文件传输工具。无需注册，四位房间码即建即用，房间关闭后一切消失。

当你需要快速传个文件给身边的人、临时共享一段文字、或者在多台设备之间倒腾东西，Flash9 让你跳过注册、扫码、加好友那些步骤——创建房间，把四位房间码发给对方，开传。房间关闭后，所有消息和文件从服务器内存中清除，不留痕迹。

---

## 快速开始

### 使用 Docker（推荐）

```bash
git clone https://github.com/vnpng/Flash9.git
cd Flash9
docker compose up -d

```

浏览器访问 `http://localhost:5017`，创建房间，把房间码发给对方，互传文字和文件。

### 原生 Node.js（需 Node.js 22+）

```bash
git clone https://github.com/vnpng/Flash9.git
cd Flash9
npm install
node server.js

```

浏览器访问 `http://localhost:8080`，创建房间，把房间码发给对方，互传文字和文件。

## 使用指南

### 创建房间

1. 打开 Flash9 首页，点击 **创建新房间**
2. 选择房间的自毁时长：1 / 2 / 3 / 5 / 7 / 9 小时（默认 1 小时）
3. 点击 **生成房间**，系统会弹出昵称设置窗口
4. 从 12 个预设趣味昵称中选一个，或自定义昵称（最长 12 字）
5. 点击 **进入房间**，你会得到一个 4 位房间码
6. 把房间码发给对方，对方输入后即可加入

房间里所有成员完全平等，没有任何角色区分。创建者并不比加入者多一些权限——谁创建的、谁先来的都无所谓，进来了就是一样的。

### 加入房间

1. 在首页输入对方给你的 4 位房间码
2. 点击 **加入**，设置昵称
3. 进入房间后自动与所有成员互通

如果输入的房间码不存在或房间已过期，页面会给出提示。

刷新页面或重新打开浏览器后，再次输入房间码加入时，会自动记住你上次使用的昵称，无需重新设置。点击左上角昵称可以随时修改。

### 发送消息

* **Enter** 发送消息
* **Shift + Enter** 换行
* 每条消息显示发送者昵称和时间（格式 HH:MM），自己的昵称为白色，他人为彩色
* 系统消息（如成员加入、退出）以灰色文字显示
* 鼠标悬停或点击消息气泡可显示 📋 复制按钮，点击即可复制消息内容
* **修改昵称后**，你之前发送的所有消息上的昵称都会同步更新

### 发送文件

三种方式任选：

* **拖拽**：把文件直接拖到输入框，边框变绿即松手
* **选择**：点击右下角 ➕ 按钮，从文件管理器中选择
* **粘贴**：在房间内按 Ctrl + V，粘贴剪贴板中的图片或文件（资源管理器中复制的文件也可以）

文件大小限制 9MB，超过会有提示。传输进度以百分比文字显示在聊天气泡中。发送方和接收方都可以点击气泡中的 **📥 下载文件** 按钮保存到本地，支持重复下载。

### 退出房间

点击 **🚪 退出房间**，确认后即可离开。其他人不受影响，继续正常聊天。

**房间的生命周期很简单**：有人在，房间就在；最后一个人离开，房间立即销毁，所有内容消失。

### 多人协作

Flash9 支持多人同时在线。所有消息和文件全部通过 WebSocket 走服务器中继，进来就能和所有人互通，不受网络环境限制。顶部栏实时显示当前在线人数。

## 快捷键

| 快捷键 | 使用场景 | 作用 |
| --- | --- | --- |
| Enter | 输入框中 | 发送消息 |

点击左上角昵称可随时修改。

## 房间码

房间码由 4 位字符组成，随机生成。字符集特意排除了容易看错的字符——没有 I、O、0、1，不用担心搞混。

进入房间后，底部栏会显示房间码胶囊。**点击即可一键复制**，方便分享给他人。

---

## 自毁机制

创建房间时选择一个自毁时长（1~9 小时）。从房间创建那一刻开始倒计时，顶部栏实时显示剩余时间。倒计时归零后：

* 所有人自动断开连接
* 房间从服务器内存中删除
* 所有消息、文件、记录全部消失

此外，**最后一个人离开房间时，房间也会立即销毁**，不会留下痕迹。

---

## 连接状态

昵称左侧有一个圆点，实时反映你与服务器的连接状态：

* 🟢 **绿色**：已连接到服务器（正常）
* 🔴 **红色**：与服务器断开（请检查网络）

所有通信全部通过 WebSocket 走服务器中继，只要绿灯亮着，文字消息和文件传输就一切正常。

## 文件传输说明

* **支持格式**：所有文件类型（图片、文档、压缩包、视频等）
* **大小限制**：9MB
* **传输通道**：全部走 WebSocket 中继，稳定可靠
* **分片大小**：16KB，逐片发送并即时显示进度
* **接收**：传输完成后不会自动下载，需点击气泡中的下载按钮手动保存
* **发送方下载**：自己发的文件自己也有下载按钮，方便在传输完成后确认或再次保存
* **剪贴板图片**：粘贴的图片如果没有文件名，会自动提取真实后缀名（未知则回退为 `.png`），命名为 `clipboard_时间戳.扩展名`
* **文件气泡**：显示发送者昵称，可点击 📋 按钮复制文件名

## 常见问题

**传输安全吗？**

Flash9 不提供端到端加密，数据在传输过程中经服务器中转。适合临时传普通文件，不要用来传输敏感信息（如密码、证件等）。

**数据存在哪里？**

所有数据只存在于服务器内存中，不写入磁盘。房间销毁或过期后数据立即清除，服务器重启也会清空所有房间。

**能用手机访问吗？**

可以。页面自动适配移动端屏幕，但手机端不支持 Ctrl+V 粘贴文件和 Ctrl+C 复制房间码（可使用长按或原生操作代替）。

**为什么对方加不进来？**

* 确认房间码输入正确（注意大小写，建议直接复制）
* 房间可能已过期（超过了自毁时间）或已被销毁
* 确认访问的端口正确（Docker 默认 5017，Node.js 默认 8080）且防火墙已放行

**连接不上 / 传输慢怎么办？**

* 所有数据通过 WebSocket 中继，速度取决于服务器带宽
* 如果双方在同一局域网，通过服务器中转通常也很快
* 检查服务器负载和网络延迟

## 部署

### Docker 部署（推荐）

项目内置 `docker-compose.yml`，默认将容器的 `8080` 端口映射到宿主机的 `5017` 端口：

```bash
docker compose up -d

```

### 基础部署（需 Node.js 22+）

将项目上传到你的服务器，运行：

```bash
node server.js

```

确保防火墙开放对应端口（默认 8080）。也可以通过环境变量自定义端口：

```bash
PORT=3000 node server.js

```

### 使用 pm2 守护

```bash
npm install -g pm2
pm2 start server.js --name flash9
pm2 save
pm2 startup

```

### 反向代理（Nginx）

如果需要绑定域名或启用 HTTPS（让 WebSocket 走 WSS 安全连接）：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:5017; # 若使用原生 Node.js 请改为 8080
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

```

`Upgrade` 和 `Connection` 头是关键——缺少这两行 WebSocket 连接会失败。

## 技术栈

* **前端**：单文件 HTML（Vanilla JS）
* **通信**：纯 WebSocket（Node.js + ws），全双工星型拓扑
* **文件传输**：WebSocket 中继（流式分片，16KB/片）
