# Flash9 ⚡

极简 · 临时 · 即焚文件传输工具。无需注册，四位房间码即建即用，房间关闭后一切消失。

当你需要快速传个文件给身边的人、临时共享一段文字、或者在多台设备之间倒腾东西，Flash9 让你跳过注册、扫码、加好友那些步骤——创建房间，把四位房间码发给对方，开传。房间关闭后，所有消息和文件从服务器内存中清除，不留痕迹。

---

## 快速开始

```bash
git clone https://github.com/vnpng/Flash9.git
cd Flash9
npm install
node server.js
```

浏览器访问 `http://localhost:8080`，创建房间，把房间码发给对方，互传文字和文件。

---

## 使用指南

### 创建房间

1. 打开 Flash9 首页，点击 **创建新房间**
2. 选择房间的自毁时长：1 / 2 / 3 / 5 / 7 / 9 小时（默认 1 小时）
3. 点击 **生成房间**，系统会弹出昵称设置窗口
4. 从 12 个预设趣味昵称中选一个，或自定义昵称（最长 12 字）
5. 点击 **进入房间**，你会得到一个 4 位房间码
6. 把房间码发给对方，对方输入后即可加入

你是房主，房间由你掌控——你在房间就在，你销毁房间所有人都会断开。

### 加入房间

1. 在首页输入对方给你的 4 位房间码
2. 点击 **加入**，设置昵称
3. 进入房间后自动与房主和其他成员建立连接

如果输入的房间码不存在或房间已过期，页面会给出提示。

### 发送消息

- **Enter** 发送消息
- **Shift + Enter** 换行
- 消息会显示你的昵称颜色、时间戳（格式 HH:MM）
- 系统消息（如成员加入/离开/重连）以灰色文字显示
- 鼠标悬停或点击消息气泡可显示 📋 复制按钮，点击即可复制消息内容

### 发送文件

三种方式任选：

- **拖拽**：把文件直接拖到输入框，边框变绿即松手
- **选择**：点击右下角 ＋ 按钮，从文件管理器中选择
- **粘贴**：在房间内按 Ctrl + V，粘贴剪贴板中的图片或文件（资源管理器中复制的文件也可以）

文件大小限制 9MB，超过会有提示。传输进度以百分比文字显示在聊天气泡中。接收方需要手动点击气泡下方的 **📥 下载文件** 按钮保存到本地，支持重复下载。

### 退出与销毁

| 操作 | 按钮 | 效果 |
|------|------|------|
| 退出房间（非房主） | 🚪 退出房间 | 你自己离开，其他人不受影响 |
| 销毁房间（房主） | 💣 销毁房间 | 所有人立即断开，房间从服务器删除，不可恢复 |

销毁前会弹出确认窗口，防止误操作。

### 多人协作

Flash9 支持多人同时在线。所有消息和文件通过服务器中继，加入后自动和所有人互通，不受网络环境限制。顶部栏实时显示当前在线人数。

## 快捷键

| 快捷键 | 使用场景 | 作用 |
|--------|----------|------|
| Enter | 输入框中 | 发送消息 |
| Shift + Enter | 输入框中 | 换行 |
| ↑ | 输入框为空时 | 回填上一条发送的消息 |
| Ctrl + V | 房间内任意位置 | 粘贴剪贴板中的图片或文件 |
| Ctrl + C | 房间内（不在输入框） | 复制房间码到剪贴板 |
| Esc | 弹窗打开时 | 关闭当前弹窗 |

房间底部有 ⌨️ 快捷键帮助按钮，点击可随时查看。

## 房间码

房间码由 4 位字符组成，随机生成。字符集特意排除了容易看错的字符——没有 I、O、0、1，不用担心搞混。

进入房间后，底部栏会显示房间码胶囊。**点击即可一键复制**，方便分享给他人。

---

## 自毁机制

房主创建房间时选择一个自毁时长（1~9 小时）。从房间创建那一刻开始倒计时，顶部栏实时显示剩余时间。倒计时归零后：

- 所有人自动断开连接
- 房间从服务器内存中删除
- 所有消息、文件、记录全部消失

房主也可以提前手动销毁房间（点击 💣 销毁房间 按钮），效果相同。

---

## 连接状态

昵称左侧有一个圆点，实时反映你与服务器的连接状态：

- 🟢 **绿色**：已连接到服务器（正常）
- 🔴 **红色**：与服务器断开（请检查网络）

如果网络波动导致与某个成员之间的连接意外断开，系统会自动尝试重连（1 秒 → 2 秒 → 4 秒……最长间隔 30 秒），重连成功后显示 ✅ 提示。不需要手动刷新页面。消息通过服务器中继，即使 WebRTC 断开，文字消息和文件传输不受影响。

## 文件传输说明

- **支持格式**：所有文件类型（图片、文档、压缩包、视频等）
- **大小限制**：9MB
- **传输通道**：全部走 WebSocket 中继，稳定可靠，不会因为防火墙或 NAT 问题卡住
- **分片大小**：16KB，逐片发送并即时显示进度
- **接收**：传输完成后不会自动下载到本地，需点击气泡中的下载按钮手动保存
- **剪贴板图片**：粘贴的图片如果没有文件名，会自动命名为 `clipboard_时间戳.png`
- **文件气泡**：显示发送者昵称，可点击 📋 按钮复制文件名

## 常见问题

**传输安全吗？**

Flash9 不提供端到端加密，数据在传输过程中经服务器中转。适合临时传普通文件，不要用来传输敏感信息（如密码、证件等）。

**数据存在哪里？**

所有数据只存在于服务器内存中，不写入磁盘。房间销毁或过期后数据立即清除，服务器重启也会清空所有房间。

**能用手机访问吗？**

可以。页面自动适配移动端屏幕，但手机端不支持 Ctrl+V 粘贴文件和 Ctrl+C 复制房间码（可使用长按或原生操作代替）。

**为什么对方加不进来？**

- 确认房间码输入正确（注意大小写，建议直接复制）
- 房间可能已过期（超过了自毁时间）
- 房主可能已经销毁了房间
- 服务器端口（默认 8080）需要防火墙放行

**连接不上 / 传输慢怎么办？**

- Flash9 使用 Google 的公共 STUN 服务器协助建立连接。确保你的网络能访问 `stun.l.google.com`
- 如果双方在同一局域网，速度通常很快
- 文件传输走 WebSocket 中继，速度取决于服务器带宽

## 部署

### 基础部署

将项目上传到你的服务器，运行：

```bash
node server.js
```

确保防火墙开放对应端口（默认 8080）。也可以通过环境变量自定义端口：

```bash
PORT=3000 node server.js
```

### 使用 pm2 守护

```bash
npm install -g pm2
pm2 start server.js --name flash9
pm2 save
pm2 startup
```

### 反向代理（Nginx）

如果需要绑定域名或启用 HTTPS（让 WebSocket 走 WSS 安全连接）：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

`Upgrade` 和 `Connection` 头是关键——缺少这两行 WebSocket 连接会失败。

## 技术栈

- **前端**：单文件 HTML（Vanilla JS）
- **信令**：WebSocket（Node.js + ws）
- **消息通信**：WebRTC DataChannel（文字消息、信令辅助）
- **文件传输**：WebSocket 中继（流式分片，16KB/片）
- **STUN 服务器**：`stun.l.google.com:19302`

---

## 浏览器兼容性

Flash9 依赖 WebRTC 和 WebSocket，需要较新版本的浏览器：

- Chrome / Edge 80+
- Firefox 75+
- Safari 14.1+（部分功能可能受限）

移动端 Chrome 和 Safari 均可使用，但粘贴文件功能在 iOS 上可能无法使用（系统限制）。
