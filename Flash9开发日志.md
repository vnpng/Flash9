# Flash9 开发日志

## 2026-05-18 — v7 纯 WebSocket 重构

### 去房主化 + 去 WebRTC
- **移除房主角色**：所有成员完全平等，退出按钮统一。不再有销毁房间功能，最后一人离开房间立即销毁
- **WebSocket 星型拓扑**：通信改为纯 WebSocket 中继，不再使用 WebRTC DataChannel
- **砍掉 STUN**：不再依赖 Google STUN 服务器
- **peer-info 改走 WS**：昵称交换通过服务端广播，注册时即完成
- **服务端简化**：register 返回已有成员列表，peer-joined 广播新人信息，去掉定向转发和信令转发

### 昵称持久化
- **localStorage 记住昵称**：刷新再入自动带出上次昵称，跳过弹窗
- **左上角点击修改**：随时改名，peer-info 广播所有人
- **改名全量更新**：修改昵称后历史消息气泡同步刷新

### 消息气泡优化
- **时间挪到昵称后面**：`👤 昵称 12:30`，不再右下角独立一行
- **自己的消息也显示昵称**：白色文字，他人彩色
- **DataChannel 参数传递**：气泡挂 data-peer-id，改名覆盖历史

### 文件传输增强
- **发送方也显示下载按钮**：自己发的文件可重复下载
- **下载按钮改黄色**：与气泡绿色区分

### 连接与生命周期
- **在线人数统计**：改回基于 WS 连接数
- **自动销毁去房主化**：任何人倒计时归零都可触发房间销毁
- **立即销毁**：最后一人离开房间即刻销毁，无缓冲

## 2026-05-16 — v4.1 修复 ICE candidate 序列化 + 进度条颜色
- **🔧 ICE candidate 序列化**：和 SDP 同类 bug，`RTCIceCandidate` 对象无法被 structured clone，改为手动拆 `{candidate, sdpMid, sdpMLineIndex}` 后传递。修复前 ICE 交换一直失败导致 DataChannel 无法打开

## 2026-05-17 — v6 跨设备 WebSocket 信令
- **WebSocket 信令服务器** (`server.js`)：Node.js + ws，同时提供 HTTP 静态文件服务
- 房间元数据从 `localStorage` 迁移到服务器内存管理
- **客户端改造**：`BroadcastChannel` 全部替换为 `WebSocket`
  - 创建房间：先生成房间码 → WS 注册 → 服务器确认 → 进入房间
  - 加入房间：输入房间码 → WS 注册 → 服务器校验房间是否存在 → 进入
  - 所有信令消息 (`offer/answer/ice-candidate/mesh-*`) 经 WS 中转
- 运行时：`node server.js` → 浏览器访问 `http://localhost:8080`

## 2026-05-17 — v7 文件传输全面改用 WebSocket 中继
- **所有文件走 WS 中继**：删除了 DataChannel 直传、ACK 流控、阈值判断，统一路径
- **流式分片**：发送端从 stream 读数据后手动按 16KB 切片，确保分片数与 `file-meta` 的 `totalChunks` 精确对齐
- **二进制消息转发**：服务端按首字节区分 JSON/二进制，二进制分片直接流式转发到房间其他人
- **过早分片暂存**：二进制分片先于 `file-meta` JSON 到达时暂存，`file-meta` 到达后补入
- **`binaryType = "arraybuffer"`**：消除 Blob 异步转换竞态
- **大文件不再断联**：走 TCP 流控替代 DataChannel SCTP 流控
- **非房主退出**：黄色 `🚪 退出房间` 按钮，发送 `peer-leave` 通知，不再触发重连
- **倒计时修复**：服务器 `register-ok` 携带 `destroyTimestamp`，所有成员同步倒计时
- **昵称弹窗恢复**：房主创建流程统一弹昵称弹窗
